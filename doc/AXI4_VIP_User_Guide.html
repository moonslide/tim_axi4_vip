Overview

The AXI4 Verification IP (VIP) is a comprehensive, production-ready UVM-based verification solution for ARM® AMBA® AXI4 protocol. It provides complete protocol compliance verification with scalable architecture supporting bus matrices from 4x4 to 64x64 and beyond.

✅ 100% Pass Rate
All 140+ tests passing with v2.5. Complete regression suite with guaranteed stability.
🚀 High Performance
Average test execution time of 7.9 seconds with parallel LSF support.
📊 Scalable Architecture
Support for any bus matrix size from 4x4 to 64x64 without code modification.

Key Features

  • Full IHI0022D AXI4 specification compliance
  • Scalable bus matrix reference model with multiple modes
  • Comprehensive QoS and USER signal support
  • Advanced protocol checking with assertions
  • Built-in error injection capabilities
  • Exclusive access transaction support
  • Out-of-order transaction handling
  • Complete functional and code coverage

Memory Implementation
class axi4_slave_memory;
    // Sparse memory implementation
    protected byte mem_array[bit[63:0]];
    
    function void write(bit[63:0] addr, byte data[]);
        foreach(data[i]) begin
            mem_array[addr + i] = data[i];
        end
    endfunction
    
    function void read(bit[63:0] addr, int size, ref byte data[]);
        data = new[size];
        foreach(data[i]) begin
            if(mem_array.exists(addr + i))
                data[i] = mem_array[addr + i];
            else
                data[i] = 8'h00; // Default value
        end
    endfunction
endclass

Routing Algorithm

Address Decode Function
function int get_slave_id(bit [63:0] addr);
    for(int i = 0; i < `NUM_SLAVES; i++) begin
        if(addr >= SLAVE_BASE_ADDR[i] && 
           addr < SLAVE_BASE_ADDR[i] + SLAVE_SIZE[i]) begin
            return i;
        end
    end
    return -1; // Invalid address - return DECERR
endfunction

Configuration Best Practices

💡 Configuration Tips
  • Always run make clean after changing configuration parameters
  • Use power-of-2 values for NUM_MASTERS and NUM_SLAVES when possible
  • Ensure ID_MAP_BITS is sufficient for your matrix size
  • Match DATA_WIDTH to your DUT's actual data bus width
  • Configure address maps to avoid overlapping regions

Configuration Validation

Configuration Checks
// Validate configuration at compile time

Exclusive Access

Exclusive Access Protocol

The VIP implements AXI4 exclusive access for atomic operations:

Exclusive Access Flow

  1. Exclusive Read: Master performs exclusive read (ARLOCK=1)
  2. Monitor Setup: Slave sets up exclusive monitor
  3. Exclusive Write: Master attempts exclusive write (AWLOCK=1)
  4. Success/Fail: Slave returns EXOKAY or OKAY response
Exclusive Access Implementation
class exclusive_monitor;
    // Track exclusive access reservations
    typedef struct {
        bit [63:0] addr;
        bit [7:0] len;
        bit [2:0] size;
        bit [3:0] master_id;
        bit valid;
    } exclusive_entry_t;
    
    exclusive_entry_t monitors[int];
    
    function void set_exclusive(int master_id, bit[63:0] addr, 
                               bit[7:0] len, bit[2:0] size);
        exclusive_entry_t entry;
        entry.addr = addr;
        entry.len = len;
        entry.size = size;
        entry.master_id = master_id;
        entry.valid = 1;
        monitors[master_id] = entry;
    endfunction
    
    function bit check_exclusive(int master_id, bit[63:0] addr);
        if(monitors.exists(master_id)) begin
            if(monitors[master_id].valid && 
               monitors[master_id].addr == addr) begin
                monitors[master_id].valid = 0; // Clear on success
                return 1; // EXOKAY
            end
        end
        return 0; // OKAY
    endfunction
endclass
